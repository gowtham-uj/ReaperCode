# documentation-writing

Write clear, accurate docs with compilable code samples.

## Steps

1. Identify the audience (user, contributor, operator).
2. State the goal in the first sentence.
3. Show, don't tell: a working code sample, then a one-line explanation.
4. Cover the failure modes users actually hit.
5. Cross-link to related pages.
6. Compile / lint the code samples; do not ship broken snippets.

## When NOT to use

- The doc is a marketing one-pager — wrong audience.
- The change is a typo in a comment.

## Validation

Every code sample must run as written (or be marked as pseudo-code).
